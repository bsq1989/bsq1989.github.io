Mac下配置opencv环境

发表于 2019-05-29 | 分类于 图像处理

OpenCV

应该不用多介绍了，平时沾点图像处理的应该都了解，开源的计算机视觉库，算法齐全，C++开发，而且运行效率高，直接用到产品中也是可以的。在这主要记录一下mac下配置opencv的过程。

OpenCV 在mac下的开发环境配置

平时的开发工作主要集中在windows平台上，怀抱微软的大腿，（虽然美国佬贸易战嘴脸丑恶，但是还是要承认某些领域的差距）,有宇宙第一编译器visual studio，在win平台开发体验还是杠杠的，配置工程各种依赖项，各种工程配置都是通过GUI去配置。为了跳出这个工作习惯，加之今年开始用git写笔记，家里的mac本用了起来，最近的工作又跟图像处理相关，因此想着能否在mac本上配置一下openCV的环境，以便平时写些demo，跑跑算法。

源代码下载

官方网站：https://opencv.org/
页面如下所示：

选择Release页，会列出所有发行版，现在版本已经比较多了，已经更新到了4.x.x，相比最早的库加入了机器学习以及深度学习相关功能模块，这次下载的为opencv4.10.0。下载到本地后，解压，然后在解压文件夹下新建一个文件夹，通常我取名build，用于存储一些构建需要的中间文件。

CMake下载

mac下编译开源代码需要cmake工具，跟win平台差不多，去cmake主页 :https://cmake.org/ 下载mac对应的dmg应用程序，并安装。

CMake构建opencv

下载完成后，在本机的profile文件中增加CMake的程序路径到PATH环境变量，在用户路径下的.bash_profile下增加也可以。

增加后，可以开始构建opencv源代码了。在mac构建代码与win上差不多，因为cmake的GUI比较完善，流程与win下相同， 打开软件，在最上面选择代码路径，也就是包含CMakeList.txt的路径，在下面一行选择生成构建配置文件的路径

然后点击下方的config按钮，开始进行配置解析，有时候会有一些警告，不用管，这里说一下，在进行源代码构建的时候，不是所有的第三方依赖都需要配置，需要根据自己的需求进行配置，具体需求就要看代码路径里的一些说明了。对于opencv的编译，config过后直接点击generate进行生成即可。
这里其实与win平台都一样，只不过在win平台上，build中生成的是vs 的sln文件，这里生成的是gcc需要的编译配置文件。
接下来就可以进行源码编译了，需要在命令行下进行，打开终端，切换到build路径，直接输入make指令，即可。然后输入sudo make install，所有的头文件以及动态库，静态库都会被拷贝至CMAKELists.txt中配置的INSTALL_PREFIX路径中，最后的make install一定要加sudo，因为一般的安装路径都会配置到系统根目录下的usr/share中，否则没有权限创建路径。在编译的过程中，遇到了个问题：
1.在这期间更新了mac系统，刚刚更新系统后，执行config/generate时，提示xcrun error,这里，需要下载mac的xcode-select。打开终端，输入
xcode-select --install
2.安装完成后，在此config,提示找不到‘sys’下的头文件，百度后，找到一个相似的问题，原理上其实是开发者的运行时环境没有安装释放相关头文件，需要前往到如下路径
/Library/Developer/CommandLineTools/Packages
在该路径下，双击Command_Line_Tools_macOS_xx_for_Xcode_xx.dmg进行安装，安装过后，即可完成config/generate过程。

OpenCV demo

这几年jetbrain公司的IDE大火，在Mac下准备使用CLion进行C++开发，jetbrain系列界面简洁，而且idea，pycharm，clion之间界面风格以及配置ui布局基本相同。
第一次转到unix/linux开发，第一个问题就是如何配置依赖项。在CLion中，这些配置均在CMakeList.txt中完成。
首先，新建工程testopenCv，在工程路径下，可看到CMakeList.txt文件，代开文件，配置如下: 

cmake_minimum_required(VERSION 3.13)
set(CMAKE_CXX_STANDARD 11)
project(testOpenCv)

#find_package(OpenCV)

include_directories(/usr/local/include/opencv4/)#配置代码文件包含路径
set(CMAKE_CXX_STANDARD 11)

add_executable(testOpenCv main.cpp)
#配置链接库
target_link_libraries(testOpenCv /usr/local/lib/libopencv_core.4.1.0.dylib)
target_link_libraries(testOpenCv /usr/local/lib/libopencv_imgcodecs.4.1.0.dylib)
target_link_libraries(testOpenCv /usr/local/lib/libopencv_highgui.4.1.0.dylib)

配置完成后，clion会自动对该脚本进行构建。配置好后，就可以进行代码编写了，测试代码如下: 

#include<iostream>
#include "opencv2/opencv.hpp"

using namespace cv;
int main() {
   Mat img=imread("./fruits.jpg");
    std::cout<<img.rows<<","<<img.cols<<std::endl;
    std::cout<<"channels:"<<img.channels();

    imshow("test",img);
    waitKey(10);
    return 0;
}

运行如下代码后，会在控制台上打印出图像宽高以及图像通道数，并弹出窗口显示图像。
